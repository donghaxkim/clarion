# LiveKit Agent Orb Implementation Plan

> **For agentic workers:** REQUIRED: Use superpowers:subagent-driven-development (if subagents available) or superpowers:executing-plans to implement this plan. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add a floating `AgentAudioVisualizerAura` orb to the bottom-left of the evidence canvas that cycles through mock LiveKit agent states on click, while moving ZoomControls to the bottom-right.

**Architecture:** Install `@livekit/components-react` + `next-themes` as npm packages, scaffold `AgentAudioVisualizerAura` via the LiveKit Agents UI shadcn registry, then wire a self-contained `AgentOrb` component that manages local mock state. No LiveKit server required — `state` is passed directly as a prop. ThemeProvider added via a thin client wrapper to avoid breaking Server Component `layout.tsx`.

**Tech Stack:** Next.js 14 (App Router), TypeScript, `@livekit/components-react@^2`, `livekit-client`, `next-themes`, shadcn agents-ui registry

**Spec:** `docs/superpowers/specs/2026-03-11-livekit-agent-orb-design.md`

## Chunk 3: Agent Orb Component

### Task 5: Create AgentOrb component

**Files:**
- Create: `experience/src/components/agent/AgentOrb.tsx`

- [ ] **Step 1: Verify AgentState enum values**

`@livekit/components-react` is ESM — do not use `require()`. Read the type definitions directly:
```bash
grep -A 10 "AgentState" experience/node_modules/@livekit/components-react/dist/index.d.ts | head -20
```

Expected output (v2.x): enum with members `Disconnected`, `Listening`, `Speaking`, `Thinking`. Also note whether the values are **numeric** (e.g. `Disconnected = 0`) or **string** (e.g. `Disconnected = 'disconnected'`) — this affects the dev label in Step 2. If names differ from expected, adjust the `CYCLE` array accordingly.

- [ ] **Step 2: Create the component**

Create `experience/src/components/agent/AgentOrb.tsx`:

```tsx
'use client';

import { useState } from 'react';
import { AgentState } from '@livekit/components-react';
import { useTheme } from 'next-themes';
// Adjust this import path if the generated entry file has a different name
import { AgentAudioVisualizerAura } from '@/components/agents-ui/agent-audio-visualizer-aura';

const CYCLE: AgentState[] = [
  AgentState.Disconnected,
  AgentState.Listening,
  AgentState.Speaking,
];

export function AgentOrb() {
  const [stateIndex, setStateIndex] = useState(0);
  const { resolvedTheme } = useTheme();

  const mockState = CYCLE[stateIndex];
  // Ternary required: resolvedTheme is string | undefined; component expects 'light' | 'dark'
  const themeMode = resolvedTheme === 'light' ? 'light' : 'dark';

  function handleClick() {
    setStateIndex((i) => (i + 1) % CYCLE.length);
  }

  return (
    <div
      role="button"
      tabIndex={0}
      onClick={handleClick}
      onKeyDown={(e) => e.key === 'Enter' && handleClick()}
      style={{ cursor: 'pointer' }}
    >
      <AgentAudioVisualizerAura
        state={mockState}
        color="#1FD5F9"
        colorShift={1.95}
        size="xl"
        themeMode={themeMode}
        className="aspect-square w-full"
      />
      {/* Dev label — remove when connecting real LiveKit */}
      {/* If AgentState is a numeric enum (e.g. Disconnected = 0), AgentState[mockState] gives "Disconnected".
          If AgentState is a string enum (e.g. Disconnected = 'disconnected'), use `mockState` directly instead. */}
      <div
        style={{
          textAlign: 'center',
          fontSize: '10px',
          color: 'var(--text-tertiary)',
          marginTop: '4px',
          fontFamily: 'DM Mono, monospace',
          userSelect: 'none',
        }}
      >
        {AgentState[mockState] ?? mockState}
      </div>
    </div>
  );
}
```

> If any prop (`colorShift`, `themeMode`, etc.) does not exist in the generated `AgentAudioVisualizerAura` source, remove it. TypeScript will error on unknown props — treat TS errors as a signal to adjust.
